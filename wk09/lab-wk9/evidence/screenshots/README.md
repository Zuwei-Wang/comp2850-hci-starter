# Screenshots Folder

Place screenshots of the interface during pilot sessions here.

## Recommended Screenshots

1. **Filter in action** (`filter-active.png`):
   - Show the filter input with "report" typed
   - Show filtered task list (should display 3 tasks)
   - Capture timestamp/status message if visible

2. **Inline edit mode** (`inline-edit-active.png`):
   - Show a task in edit mode
   - Show the input field with text
   - Show Save/Cancel buttons
   - Include focus indicator if visible

3. **Delete confirmation modal** (`delete-confirmation.png`):
   - Show the confirmation dialog
   - Show task to be deleted highlighted/mentioned
   - Show "Confirm" and "Cancel" buttons

4. **Validation error** (`validation-error.png`):
   - Show error message for blank title
   - Show error styling (red border, ARIA alert, etc.)
   - Include the form context

5. **Successful task addition** (`task-added-success.png`):
   - Show newly added task in list
   - Show success message if visible
   - Include timestamp/status indicator

6. **Pagination controls** (`pagination.png`):
   - Show page numbers and navigation
   - Show disabled/active state styling

## Guidelines

- **Resolution**: Use high enough resolution to show details (at least 1280x720)
- **Redaction**: If any test data contains sensitive info, redact before saving
- **Annotations**: You can add arrows/highlights in an image editor if helpful
- **File format**: PNG or JPEG (PNG preferred for UI screenshots)
- **File size**: Keep under 2MB per image for easy sharing

---

**Note**: Screenshots are for documentation and evidence purposes. Do NOT capture any participant's personal information (names, faces, etc.)
